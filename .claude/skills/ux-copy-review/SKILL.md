# UX Copy Review

Атомарный скилл: ТОЛЬКО тексты интерфейса.
Для визуальной эстетики и layout → прочитай `.claude/skills/frontend-design/SKILL.md` через Read tool.
Для доступности (WCAG) → прочитай `.claude/skills/accessibility-audit/SKILL.md` через Read tool.

## Два режима работы

### Режим 1: Спецификация (PLAN, IMPL)

Создание текстов для нового UI. Результат — спецификация текстов для Design Document секция 6b или готовые строки для компонентов.

### Режим 2: Проверка (AUDIT, OPT)

Валидация существующих текстов в коде. Результат — список нарушений с severity и предложениями замены.

---

## Шаг 1: Загрузка контекста

Прочитай оба файла:
1. `docs/design/UX-WRITING-GUIDE.md` — copy SSOT: правила UX-текстов, режимы тона, глоссарий
2. `docs/design/DESIGN-SYSTEM.md` — visual SSOT + секция 14 как quick-reference examples

## Шаг 2: Определи режим тона

Какой контекст у экрана? От этого зависит обращение и характер текста:

| Режим | Обращение | Характер |
|-------|-----------|----------|
| System UI (навигация, ошибки) | ты | Дружелюбный, неформальный |
| Onboarding (первый запуск) | ты | Тёплый, ободряющий |
| Published docs (сайт документации) | вы | Нейтральный, профессиональный |
| AI-ассистент (внутренний) | ты | Дружелюбный + компетентный |
| AI-ассистент (публичный) | вы | Дружелюбный + компетентный |
| Enterprise / compliance | вы | Формальный, точный |

## Шаг 3: Сбор текстов

Найди все user-facing строки:
- Статический текст в JSX (`<Button>Текст</Button>`)
- Placeholder, aria-label, title, alt атрибуты
- Toast/notification сообщения (Sonner)
- Error messages (в schema.ts, service.ts)
- Empty state текст
- AI chat сообщения

## Шаг 4: Проверка по чеклисту

- [ ] **Тон:** соответствует режиму (см. таблицу выше)
- [ ] **Кнопки:** глагол в инфинитиве, конкретный ("Создать документ", не "Ок" или "Действие")
- [ ] **Ошибки:** формула [что случилось] + [что делать], без кодов (500, 403)
- [ ] **Empty states:** icon + heading + description + CTA
- [ ] **Tooltips:** ≤80 символов, без точки если одно предложение
- [ ] **Терминология:** по глоссарию (Пространство, Документ, Секция — не workspace, page, section)
- [ ] **Форматирование:** sentence case, правила пунктуации
- [ ] **Консистентность:** одинаковые термины для одинаковых концепций
- [ ] **Нет placeholder-текстов:** "Lorem ipsum", "TODO: text"
- [ ] **Нет технических кодов:** HTTP статусы, stack traces, внутренние ID

### Примеры

| Элемент | Плохо | Хорошо |
|---------|-------|--------|
| Кнопка | "Ок", "Подтвердить", "Действие" | "Создать документ", "Сохранить изменения" |
| Ошибка | "Error 500", "Forbidden" | "Не удалось сохранить. Попробуй через минуту." |
| Empty state | "Нет данных" | "Пока пусто — самое время создать первый документ." |
| Tooltip | "Данная функция позволяет осуществить создание нового пространства" | "Создать новое пространство" |
| Confirmation | "Вы уверены?" | "Удалить документ «Гайд»? Это действие нельзя отменить." |
| AI greeting | "Здравствуйте, чем могу помочь" | "Привет! Задай вопрос по документации." |

## Шаг 5: Классификация по severity

| Severity | Когда | Пример |
|----------|-------|--------|
| `CRITICAL` | Пользователь видит технический текст, нарушена терминология | "Error 500", "workspace" вместо "Пространство" |
| `MAJOR` | Тон не соответствует режиму, кнопка без глагола | Канцелярит в онбординге, "Подтвердить" вместо "Создать" |
| `MINOR` | Стилистика, длина tooltip, пунктуация | Точка в заголовке, tooltip >80 символов |

## Шаг 6: Вердикт

- **PASS** — все тексты соответствуют UX-WRITING-GUIDE.md
- **SUGGESTIONS** — есть рекомендации (MAJOR/MINOR)
- **FAIL** — есть CRITICAL нарушения

## Формат отчёта

В AGENT_REPORT → `uxExpert.warnings[]`:

```json
{
  "type": "copy",
  "severity": "CRITICAL|MAJOR|MINOR",
  "file": "src/features/auth/components/login-form.tsx",
  "line": 42,
  "text": "Error 500",
  "problem": "Код ошибки виден пользователю",
  "suggestion": "Что-то пошло не так. Попробуй через минуту."
}
```

**Gate mapping (Phase B):**
- `severity: CRITICAL` → включить в `blockingIssues` (триггерит FIX цикл)
- `severity: MAJOR/MINOR` → только `warnings[]` (не блокируют)
