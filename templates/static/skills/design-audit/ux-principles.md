# UX Principles — Reference Guide

Краткая теоретическая база для обоснования UX findings на продуктовых экранах.

---

## 1. Scanning Patterns

- F-pattern для текстовых и списочных layouts
- Z-pattern для промо и landing layouts
- critical info должна быть там, куда глаз идёт первой волной

## 2. Information Scent

Пользователь должен понимать, что произойдёт после действия, до клика.

## 3. Progressive Disclosure

- Tier 1: видно сразу
- Tier 2: раскрывается в один шаг
- Tier 3: живёт на отдельном экране или в отдельном flow

Часто используемое не должно жить глубоко.

## 4. Cognitive Load

- меньше одновременных решений
- разумные defaults
- понятные группы, а не длинные плоские списки

## 5. Gestalt

- proximity
- similarity
- continuity
- figure-ground

Используй эти принципы, чтобы объяснять, почему экран читается хорошо или плохо.

## 6. Feedback

Важные изменения состояния должны быть замечаемыми: success, error, loading, pending, conflict.

## 7. Overview -> Filter -> Detail

Сначала обзор, потом фильтр или навигация, потом детали по запросу.
