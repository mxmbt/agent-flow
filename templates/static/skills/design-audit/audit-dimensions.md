# Audit Dimensions — Reference Guide

8 измерений UX-аудита для продуктовых интерфейсов. Дополняет checklist из `SKILL.md`.

---

## 1. Information Architecture

- Можно ли быстро найти основную задачу?
- Группировка совпадает с ментальной моделью пользователя?
- Связанные вещи находятся рядом?

## 2. Visual Hierarchy

- Самое важное выделено первым?
- Secondary элементы не спорят с primary контентом?
- Error и warning состояния визуально заметны?

## 3. Interaction Clarity

- Что кликабельно и что произойдёт после клика?
- Есть ли предсказуемые hover/focus/pressed состояния?
- Primary action очевиден?

## 4. Progressive Disclosure

- Часто нужная информация видна сразу?
- Сложность раскрывается постепенно, а не свалена сразу?
- Детали доступны по запросу, но не спрятаны слишком глубоко?

## 5. Cognitive Load

- Не слишком ли много решений на одном экране?
- Есть ли sensible defaults?
- Повторяющиеся действия автоматизированы или упрощены?

## 6. Accessibility

- Семантика, focus, keyboard reachability, readable contrast
- Ошибки и состояния понятны не только визуально

## 7. Feedback and Change Awareness

- Пользователь замечает важные изменения состояния?
- Есть подтверждение успешных и неуспешных действий?

## 8. Content and Copy

- Тексты ведут к действию, а не описывают интерфейс канцеляритом
- Термины консистентны по всему surface
