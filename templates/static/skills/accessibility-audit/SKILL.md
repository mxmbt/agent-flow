# Accessibility Audit

Use browser inspection and interaction to check accessibility on the changed surface.

## Minimum Audit Areas

- heading structure
- accessible names for controls
- image alt text
- landmark regions
- focus behavior and keyboard reachability
- error text clarity

## Reporting

For each issue, capture:

- affected element or screen
- why it hurts accessibility
- WCAG area when clear
- visible evidence or reproduction notes

Treat accessibility defects as user-facing bugs, not polish.
